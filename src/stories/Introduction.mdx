# Autocare — Component Library

Welcome to the **Autocare** Storybook. This is the living documentation for every
component and page in the application.

---

## How to use this Storybook

Use the sidebar on the left to browse components and pages. Each entry has:

- **Stories** — interactive examples that show the component in different states.
- **Docs** — auto-generated API documentation (props, types, descriptions).

---

## Writing documentation with MDX

Every new component or page **must** include an MDX documentation file alongside
its story file. Place the MDX file next to the component:

```
src/components/
  MyComponent.tsx
  MyComponent.stories.tsx
  MyComponent.mdx          ← add this
```

### MDX file template

```mdx
{/* src/components/MyComponent.mdx */}

import { Meta, Canvas, Controls } from "@storybook/addon-docs/blocks";
import * as MyComponentStories from "./MyComponent.stories";

<Meta of={MyComponentStories} />

# MyComponent

One-paragraph description of what this component does and when to use it.

## Usage

<Canvas of={MyComponentStories.Default} />
<Controls of={MyComponentStories.Default} />

## Design notes

- Key design decisions and constraints.
- Accessibility considerations.
- Links to relevant Figma frames if applicable.
```

---

## Project conventions

- Components live in `src/components/` and are re-exported from `src/components/index.ts`.
- Pages live in `src/pages/` and are re-exported from `src/pages/index.ts`.
- Use the `~/` alias to import from `src/` (e.g. `~/components`, `~/types`).
- Styling uses MUI only — no Tailwind, no custom CSS.
- Shared Storybook utilities (theme decorator, mock fixtures) live in `src/storybooks/index.ts`.
